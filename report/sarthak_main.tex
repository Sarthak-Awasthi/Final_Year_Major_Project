\frontmatterheading{ABSTRACT}
\addcontentsline{toc}{chapter}{Abstract}

This report presents a multi-agent reinforcement learning framework for emergent cooperative behavior among NPC agents in a medieval village RPG research playground. The core contribution is the design and evaluation of a decomposed reward model with a dynamic $\lambda$ coefficient that creates a feedback loop between individual cooperation tendency and community welfare weighting: $R_i(t) = P_i(t) + G_i(t) + \lambda_i(t) \cdot C(t)$, where $\lambda_i = 0.05 + \text{coop}_i \times 0.55$. Six tabular Q-learning agents operate over a 225-state discretized space with role-specific action masking and adaptive personality dynamics. A system shock engine injects non-stationary perturbations (famine, plague, raids) to test learning robustness under distribution shift. We conduct an ablation study across six conditions over 500 turns each, demonstrating that: (1)~Q-learning drives cooperation from a 0.000 baseline to 0.699 while schedule-only agents show no emergence; (2)~role-specific action masking improves cooperation by 0.161 through structural inductive bias, the largest single contributor after RL itself; (3)~the famine shock has a substantial, lasting effect --- removing shocks raises final cooperation by 0.237 (to 0.936) --- yet agents keep learning rather than collapsing; (4)~the dynamic $\lambda$ feedback loop amplifies cooperation by a further 0.104 over a static baseline; and (5)~adaptive personality coefficients produce meaningful role divergence in social sensitivity and risk aversion. The system additionally supports optional LLM narration with template fallbacks, and the game environment is built on a hierarchical MDP for quest progression.

\chapter{Introduction}
\label{chap:introduction}

Non-player characters (NPCs) in modern games typically follow scripted behavior trees or finite state machines, producing predictable and repetitive interactions. This project explores whether reinforcement learning can produce emergent cooperative behavior in a multi-agent game environment without explicit scripting of cooperation.

We build a single-player medieval village RPG where six NPCs learn autonomously via tabular Q-learning. The central research question is:

\begin{quote}
\textit{Can NPCs learn to shift from individually optimal strategies to cooperative behavior when community welfare contributes to their total reward?}
\end{quote}

\section{Problem Statement}
\label{sec:problem-statement}

Current NPC systems in games rely on hand-authored behavior trees, finite state machines, or scripted dialogue. These approaches produce deterministic, repetitive behavior that breaks immersion and cannot adapt to player strategy. The core problems are:

\begin{itemize}
    \item \textbf{No emergent cooperation:} Scripted NPCs cannot discover cooperative strategies on their own --- any prosocial behavior must be explicitly programmed by designers for every scenario.
    \item \textbf{Brittleness to perturbation:} Static behavior trees cannot adapt when the game environment changes (e.g., resource shocks, new player strategies). NPCs continue executing their scripts regardless of context.
    \item \textbf{No individual--community trade-off:} Real social agents must balance personal gain against group welfare. Scripted NPCs have no mechanism to represent, let alone dynamically adjust, this trade-off.
    \item \textbf{Lack of interpretability in deep RL alternatives:} While deep RL can produce adaptive NPC behavior, the learned policies are opaque. For a research playground, full interpretability of what agents have learned is essential.
\end{itemize}

\section{Project Significance}
\label{sec:project-significance}

This project contributes to both game AI and multi-agent reinforcement learning research:

\begin{itemize}
    \item \textbf{Emergent social dynamics:} Demonstrates that cooperation can arise purely from reward structure and learning, without being scripted --- a result relevant to computational social science and mechanism design.
    \item \textbf{Interpretable multi-agent RL:} By using tabular Q-learning (6,300 Q-values per NPC), every learned policy is directly inspectable. This contrasts with deep RL approaches where policy interpretation requires post-hoc explanation methods.
    \item \textbf{Decomposed reward framework:} The $R_i(t) = P_i(t) + G_i(t) + \lambda_i(t) \cdot C(t)$ formulation with dynamic $\lambda$ is domain-agnostic and transferable to industrial multi-agent coordination problems (factory automation, automotive ECU tuning).
    \item \textbf{Research playground:} The system provides a complete experimental platform with ablation controls, analytics dashboard, deterministic seeding, and automated bot playthroughs for reproducible research.
\end{itemize}

\section{Scope of Work}
\label{sec:scope}

The system is structured in three layers:

\begin{itemize}
    \item \textbf{Hierarchical MDP:} A 7-stage quest graph with 16 static checkpoints drives game progression. Dynamic checkpoints handle player deviations with nudging and convergence mechanisms.
    \item \textbf{Q-Learning NPC Agents:} Six NPCs run tabular Q-learning over a 225-state $\times$ 28-action space with role-specific action masking and adaptive personality coefficients.
    \item \textbf{Optional LLM Narration:} An HTTP adapter connects to OpenRouter, Ollama, or any OpenAI-compatible provider. Every LLM call has a deterministic template fallback, ensuring the game runs identically without a model.
\end{itemize}

The project is implemented as a web application (Python/FastAPI backend, vanilla JavaScript frontend) with a real-time analytics dashboard for monitoring cooperation curves, reward decomposition, and shock response.

\section{Objectives}
\label{sec:objectives}

The main objectives of this project are:

\begin{enumerate}
    \item Design and implement a multi-agent reinforcement learning environment within a medieval village RPG setting.
    \item Develop a decomposed reward model with a dynamic $\lambda$ coefficient that creates feedback loops between individual and community welfare.
    \item Implement six tabular Q-learning NPC agents with role-specific action masking and adaptive personality dynamics.
    \item Build a system shock engine to test agent resilience under non-stationary environmental perturbations.
    \item Conduct a systematic ablation study isolating the contribution of each subsystem to emergent cooperation.
    \item Integrate optional LLM narration with guardrails and template fallbacks for atmospheric game text.
\end{enumerate}

\section{Novelty}
\label{sec:novelty}

The key novel contributions of this project are:

\begin{itemize}
    \item \textbf{Dynamic $\lambda$ feedback loop:} Unlike static reward weighting in prior MARL work, the $\lambda_i(t) = 0.05 + \text{coop}_i \times 0.55$ mechanism creates a virtuous cycle where cooperation self-amplifies through the reward structure. Our ablation shows this contributes $+0.104$ to cooperation --- one of the largest factors, alongside role masking, after RL itself.
    \item \textbf{Fully interpretable multi-agent cooperation:} While prior game AI research uses deep RL (opaque policies), our tabular approach lets researchers directly inspect every Q-value and trace exactly why an NPC chose a cooperative action.
    \item \textbf{Integrated research playground:} The system combines RL agents, environmental shocks, adaptive personalities, quest progression, LLM narration, and a complete analytics dashboard in a single reproducible platform with ablation controls --- a combination not found in existing game AI testbeds.
\end{itemize}

\chapter{Literature Survey}
\label{chap:literature}

This chapter reviews prior work across four domains that converge in our project: multi-agent reinforcement learning, reward shaping, NPC intelligence in games, and LLM-enhanced narration. For each category, we summarize the major approaches and identify the gaps our project addresses.

\section{Multi-Agent Reinforcement Learning}
\label{sec:marl}

The foundations of reinforcement learning were established by Watkins and Dayan~\cite{watkins1992qlearning}, who proved that tabular Q-learning converges to optimal policies in single-agent MDPs under standard conditions (all state-action pairs visited infinitely often, appropriate learning rate decay). Sutton and Barto~\cite{sutton2018rl} provide a comprehensive treatment of value-based methods, policy gradient approaches, and temporal-difference learning that underpin modern RL research.

Extending RL to multi-agent settings introduces fundamental challenges. Busoniu et al.~\cite{busoniu2008marl} survey the landscape of multi-agent reinforcement learning, identifying three key difficulties: (i)~the environment becomes non-stationary from each agent's perspective because other agents are simultaneously learning and changing their policies; (ii)~the joint action space grows exponentially with the number of agents; and (iii)~credit assignment --- determining which agent's action caused a team reward --- is ambiguous. Bowling and Veloso~\cite{bowling2002multiagent} address the convergence problem by proposing a variable learning rate that adapts based on whether an agent is ``winning'' or ``losing,'' providing convergence guarantees in two-player games that do not extend trivially to the $N$-agent setting.

The simplest multi-agent approach is independent learners (IL), where each agent runs its own single-agent algorithm while treating other agents as part of the environment. Despite the non-stationarity violation, IL works surprisingly well in practice when agents change slowly relative to their learning rate~\cite{busoniu2008marl}. Our system adopts this independent-learner architecture: each NPC runs its own tabular Q-learning instance. The slow adaptation rate (0.018) and homeostatic drift toward 0.5 act as implicit regularisers that keep the environment approximately stationary from each agent's perspective.

Cooperative MARL has been studied through more sophisticated frameworks. QMIX~\cite{rashid2018qmix} factorizes joint Q-values under a monotonicity constraint, enabling centralized training with decentralized execution for cooperative tasks. MADDPG~\cite{lowe2017maddpg} extends actor-critic methods to multi-agent settings by giving each agent's critic access to all agents' observations during training. Leibo et al.~\cite{leibo2017multi} study emergent social dilemmas in grid-world environments, showing that cooperation or defection can arise from independent learners depending on environmental incentives.

Our approach uses independent Q-learners with a shared community reward signal, closest to the emergent cooperation framework of Leibo et al., but with a critical addition: the dynamic $\lambda$ coefficient that gates how much each agent weights the community signal based on its own cooperation history. This mechanism provides an implicit coordination channel without requiring the centralized critics of QMIX or MADDPG.

\section{Reward Shaping}
\label{sec:reward-shaping}

Ng et al.~\cite{ng1999rewardshaping} proved that potential-based reward shaping preserves optimal policies while accelerating learning. This foundational result ensures that adding a shaping term $F(s, s') = \gamma \Phi(s') - \Phi(s)$ does not change what the agent ultimately learns, only how fast it gets there. The key insight is that the shaping reward must be defined as a difference of potentials to avoid introducing spurious equilibria.

Our decomposed reward model extends this idea in a specific direction: the community component $\lambda_i(t) \cdot C(t)$ acts as a shaping signal that nudges agents toward prosocial behavior, with the dynamic $\lambda$ controlling the strength of this nudge based on each agent's cooperation trajectory. While our shaping is not strictly potential-based (the $\lambda$ coefficient itself evolves with the agent's state, making the shaping function non-stationary), the practical effect is similar: agents that begin cooperating receive increasingly strong gradient signals toward further cooperation through the $\lambda$ feedback loop, accelerating convergence to cooperative equilibria. The ablation result --- C7 (static $\lambda$) reaches 0.595 vs.\ C1 (dynamic $\lambda$) at 0.699 --- quantifies the acceleration that this adaptive shaping provides over a fixed shaping baseline.

\section{NPC Intelligence in Games}
\label{sec:npc-intelligence}

Traditional NPC behavior in commercial games relies on behavior trees~\cite{isla2005bt}, finite state machines, or goal-oriented action planning, requiring designers to manually author every decision branch. Despite decades of RL research, adoption in shipped commercial games remains slow~\cite{liu2025schola}. Jacob et al.~\cite{jacob2020unwieldy}, based on interviews with 17 game agent creators from AAA and indie studios, document that the industry avoids RL due to unpredictable trained behavior, long training times, and insufficient tooling. Yannakakis and Togelius~\cite{yannakakis2018gameai} note that ``commercial games have largely ignored the use of learning AI.'' The rare exceptions apply learning to narrow, constrained subtasks (goalkeeper positioning, driving style modeling), not to general NPC decision-making or social behavior.

Recent academic work has explored RL for game NPCs~\cite{riedl2012interactive}, but primarily for single-agent scenarios. To the best of our knowledge, no prior work applies \textit{cooperative} multi-agent RL with dynamic reward weighting to a full game environment featuring quest progression, social dynamics among multiple NPCs, and environmental perturbations.

\section{LLM-Enhanced Game Narration}
\label{sec:llm-narration}

Large language models have been applied to interactive fiction~\cite{urbanek2019light} and game dialogue generation. Our system uses LLMs as an optional narration layer with a 9-stage guardrail pipeline, maintaining full playability without model access --- separating game logic from narration.

\chapter{Methodology}
\label{chap:methodology}

This chapter details the technical design of the system, with emphasis on the reinforcement learning architecture: the Q-learning formulation, decomposed reward model with dynamic $\lambda$, action masking, adaptive personality dynamics, and the system shock engine that tests learning robustness. The hierarchical MDP for quest progression and the LLM integration layer are covered more briefly.

\section{Hierarchical MDP}
\label{sec:hierarchical-mdp}

The quest is modeled as a two-level hierarchical MDP. At the macro level, seven stages drive game progression:

\begin{equation}
    S_1 \to S_2 \to S_3 \to S_4 \to S_5 \to S_6 \to S_7 \to S_{\text{success}}
\end{equation}

Each macro stage $S_k$ contains a micro MDP $\mathcal{M}_{\text{micro}}^{(k)}$ with its own checkpoints, completion conditions, and transition rewards. The 7 stages span: arrival at the gate, seeking the elder, learning of the quest, investigation, retrieval, and return.

When the player deviates from the expected path, the system generates dynamic checkpoints that acknowledge the deviation and guide convergence back to the nearest static checkpoint. When the quest completes before the turn limit, the quest state resets while preserving all NPC Q-tables and adaptation states, enabling multi-episode learning within a single session. This quest auto-restart mechanism is critical for the RL results: it allows NPCs to accumulate learning experience across multiple quest cycles within a single 500-turn experiment.

\label{sec:q-learning}

Each NPC maintains a tabular Q-table over a discretized state space:

\begin{tabular}{lll}

Total state space: $5 \times 5 \times 3 \times 3 = 225$ states. Action space: 28 universal actions across 7 categories (navigation, exploration, talk, social, combat, stealth, utility). Q-table size per NPC: 6,300 entries.

\textbf{Q-Learning update:}
\begin{equation}
    Q(s, a) \leftarrow Q(s, a) + \alpha \left[ r + \gamma \max_{a'} Q(s', a') - Q(s, a) \right]
\end{equation}
with $\alpha = 0.1$, $\gamma = 0.9$, $\varepsilon$-greedy exploration decaying from 0.15 to 0.05 at rate $0.995$/turn. Pre-training: 100 episodes $\times$ 50 turns per NPC at $\varepsilon = 0.5$.

\textbf{Why tabular Q-learning over deep RL.}
The choice of tabular Q-learning over function approximation methods such as DQN is deliberate and motivated by three factors. First, \emph{interpretability}: with 6,300 Q-values per NPC, the entire learned policy can be directly inspected --- researchers can look up the Q-value for any state-action pair and trace exactly why a particular action was preferred. Deep RL policies, by contrast, require post-hoc explanation methods (saliency maps, SHAP values) that provide approximate rather than exact attribution. Second, \emph{convergence guarantees}: Watkins and Dayan~\cite{watkins1992qlearning} proved that tabular Q-learning converges to the optimal policy under standard conditions. While the multi-agent setting violates the stationarity assumption, the slow adaptation rate and homeostatic drift provide empirical stability, as confirmed by the consistent convergence to $\mathcal{K} \approx 0.699$ across runs. Third, \emph{state space tractability}: the 225-state space is small enough that every state-action pair can be visited many times during a 500-turn experiment with quest auto-restart, ensuring that Q-values are well-estimated rather than interpolated.

\textbf{Epsilon-greedy exploration strategy.}
The exploration schedule uses $\varepsilon$-greedy action selection with a decaying exploration rate: $\varepsilon_t = \max(0.05,\; 0.15 \times 0.995^t)$. This schedule balances exploration and exploitation across the experiment timeline. The initial $\varepsilon = 0.15$ provides sufficient exploration to discover cooperative strategies in early turns, while the exponential decay ensures that by mid-experiment ($t \approx 200$), the exploration rate has dropped to approximately 0.055, allowing agents to exploit their learned policies. The floor of 0.05 maintains a small probability of random action throughout the experiment, preventing premature convergence to suboptimal local equilibria. During pre-training, a much higher $\varepsilon = 0.5$ ensures broad coverage of the state-action space, giving each NPC a reasonable initial Q-table before the main experiment begins.

\section{Decomposed Reward Model}
\label{sec:reward-model}

Each NPC $i$ receives a reward signal composed of three terms:
\begin{equation}
    R_i(t) = P_i(t) + G_i(t) + \lambda_i(t) \cdot C(t)
    \label{eq:reward}
\end{equation}

\begin{itemize}[nosep]
    \item $P_i(t)$: \textbf{Penalty} --- $-10$ when HP drops below 20\%.
    \item $G_i(t)$: \textbf{Individual reward} --- $\sum_k w_k \cdot (\text{stat}_k^{\text{new}} - \text{stat}_k^{\text{old}})$, weighted by archetype-specific preferences (e.g., farmers weight income at 0.4, guards weight reputation at 0.4).
    \item $C(t)$: \textbf{Community reward} --- Village-level welfare computed as $C(t) = 0.5 \cdot C_{\text{abs}}(t) + 0.5 \cdot C_{\Delta}(t)$, using centered non-linear shaping (sigmoid for reputation and mood, sub-linear power for health) that can go negative when village welfare drops.
    \item $\lambda_i(t)$: \textbf{Dynamic lambda} --- $\lambda_i = 0.05 + \text{coop}_i \times 0.55$, ranging from 0.05 (selfish) to 0.60 (cooperative). Creates a feedback loop: positive community reward $\to$ cooperation rises $\to$ $\lambda$ increases $\to$ more weight on community reward.
\end{itemize}

\textbf{Intuition behind the three components.}
The decomposition into $P$, $G$, and $\lambda \cdot C$ is designed to capture three distinct pressures that real social agents face. The penalty term $P_i(t)$ acts as a hard survival constraint: no matter how cooperative an agent becomes, it must avoid actions that endanger its own existence. This prevents degenerate cooperation where agents sacrifice themselves for the group. The individual reward $G_i(t)$ captures role-specific self-interest: a farmer that never improves its income is failing at its primary function, regardless of village welfare. The community term $\lambda_i(t) \cdot C(t)$ introduces the social dimension, but critically, it is \emph{gated} by the agent's own cooperation history via $\lambda$. This means the community signal is not imposed uniformly; agents that have historically cooperated receive a stronger community gradient, while agents that have not cooperated remain primarily self-interested. The interaction between these three terms creates a natural tension: $G_i$ pulls agents toward individually optimal actions, $\lambda_i \cdot C$ pulls toward prosocial actions, and $P_i$ provides a floor below which no agent will go. The dynamic $\lambda$ resolves this tension over time by amplifying whichever direction the agent has been moving.

\section{Role-Specific Action Masking}
\label{sec:action-masking}

Each NPC archetype has a preferred action set. During action selection:
\begin{itemize}[nosep]
    \item \textbf{Hard masking}: Invalid actions get $Q(s,a) = -\infty$.
    \item \textbf{Soft masking}: Role-aligned actions receive a $+0.5$ Q-value bonus; misaligned actions receive a $-0.3$ penalty.
\end{itemize}

The soft masking operates as an inductive bias that shapes the exploration process without eliminating it. Consider a guard NPC at the gate: combat and patrol actions receive the $+0.5$ bonus, making them more likely to be selected during $\varepsilon$-greedy exploration. However, the agent can still discover that, say, a social action yields high community reward in certain states --- the $-0.3$ penalty is small relative to the Q-value range and can be overcome by strong reward signals. The ablation result (C6 vs.\ C1, $\Delta = -0.161$) confirms that this masking accelerates learning by reducing the effective exploration space, producing cleaner Q-value estimates with less noise from irrelevant actions --- indeed, masking is the largest single contributor to cooperation after RL itself.

\textbf{Coefficient} & \textbf{Drives} & \textbf{Role Multipliers} \\
\midrule
cooperation\_tendency & $\lambda$ (community weight) & Gated by $\lambda$: $r_{\text{coop}} = r_{\text{base}} \times (0.5 + \lambda_i)$ \\
risk\_aversion & Risk avoidance & Guard 1.5$\times$, Farmer 1.2$\times$, Elder 0.6$\times$ \\
social\_sensitivity & Social feedback & Elder 1.4$\times$, Tavern 1.2$\times$, Guard 0.5$\times$ \\
shock\_resilience & Shock adaptation & Increases under shock pressure \\
\bottomrule
\end{tabular}
\end{table}

\textbf{Homeostatic drift as a stability mechanism.}
The homeostatic drift rate of 0.004 toward 0.5 serves a critical role in preventing runaway coefficient dynamics. Without it, the $\lambda$ feedback loop could drive cooperation\_tendency to its maximum (1.0) and lock agents into inflexible cooperative behavior, or conversely, a sequence of negative shocks could drive it to its minimum (0.0) and permanently suppress cooperation. The drift acts as a slow mean-reversion force: at each turn, each coefficient is pulled 0.004 toward the neutral midpoint of 0.5. This means that sustained positive or negative pressure is required to maintain extreme coefficient values. If the environmental pressure disappears (e.g., after a shock expires), the coefficients gradually return to baseline, restoring the agent's capacity to respond to future perturbations. The interaction between the adaptation rate (0.018) and the drift rate (0.004) creates a net rate of approximately 0.014 when the agent receives consistent signals, slow enough to avoid oscillation but fast enough to produce meaningful divergence over 500 turns.

\label{sec:shock-engine}

Shocks directly pressure cooperation downward: when shock intensity exceeds 0.1, cooperation tendency decreases proportionally, independent of the reward signal.

\textbf{Non-stationarity and Q-learning robustness.}
System shocks create a specific challenge for Q-learning convergence. The convergence proof of Watkins and Dayan~\cite{watkins1992qlearning} assumes a stationary MDP, but shocks temporarily alter the reward function (via the reward scale factor) and the transition dynamics (via stat drains and trust modifiers). This means Q-values learned before the shock become partially stale during it. However, the tabular Q-learning update is inherently local --- it modifies only the $Q(s,a)$ entry for the visited state-action pair --- and the learning rate $\alpha = 0.1$ allows gradual correction of stale values. The empirical result (C5 vs.\ C1, $\Delta = +0.237$) shows that the famine shock has a substantial and lasting effect on learning: it produces a visible dip in the community-reward signal at turn 166 (\cref{fig:reward}) and lastingly depresses cooperation relative to the shock-free condition (C5 reaches 0.936 vs.\ C1's 0.699). Crucially, the agents do not collapse --- the local Q-update and the shock\_resilience coefficient (which increases under shock pressure) keep cooperation rising after the shock expires --- but they do not recover the lost ground within the 500-turn session, so a single famine leaves a persistent, compounding imprint through the $\lambda$ feedback loop.

\section{LLM Integration}
\label{sec:llm-integration}

The LLM service is a thin HTTP adapter supporting Ollama, OpenRouter, and OpenAI-compatible providers. It serves four use cases: input parsing, NPC dialogue, checkpoint generation, and narration enhancement. A 9-stage guardrail pipeline (strip reasoning blocks, parse JSON, schema validation, enum validation, value clamping, action reference validation, content filtering, length enforcement) ensures output quality. Every code path has a deterministic template fallback, so the RL experiments run identically with or without an LLM. Default model: \texttt{qwen3:4b} via LM Studio or \texttt{meta-llama/llama-3.2-3b-instruct:free} via OpenRouter.

\section{Turn Processing Pipeline}
\label{sec:turn-pipeline}

Each player action triggers a 10-step pipeline:
\begin{enumerate}[nosep]
    \item Resolve player action (combat, social, navigation, etc.)
    \item Log event with importance scoring
    \item Check quest completion or deviation
    \item Process 6 NPC turns (Q-learning action selection, reward computation, Q-table update)
    \item Check random events
    \item Shock engine tick (decay intensities, apply stat drain, trust modifiers)
    \item Time advancement (4 turns = 1 period)
    \item Reputation decay, stamina regeneration, perception checks
    \item Game-over conditions (or quest restart if turns remain)
    \item Telemetry logging (world snapshot, reward decomposition, adaptation state)
\end{enumerate}

\subsection{Metrics}

\begin{itemize}[nosep]

\end{itemize}

\subsection{Finding 1: RL Drives Cooperation}

The most significant result: C1 (Full System) achieves a cooperation index of 0.699 while C3 (No RL) remains at exactly 0.000 --- the initialization baseline, since cooperation tendency starts at zero. Without Q-learning, NPCs follow deterministic schedules and cannot adapt their behavior. The 0.699 gap demonstrates that cooperation is an \textit{emergent} property of the learning process, not a byproduct of game mechanics --- RL is by far the dominant driver.

\subsection{Finding 2: Role Masking Aids Learning}

C6 (No Role Masking) achieves 0.538, compared to C1's 0.699 --- a 0.161 difference, the largest single-component contribution to cooperation after RL itself. Without role constraints, NPCs explore the full 28-action space uniformly, producing noisier Q-value estimates. Role masking provides structural inductive bias: guards learn from guard-relevant actions, farmers from farming actions, producing cleaner gradient signals.

\subsection{Finding 3: Shock Resilience}

C5 (No Shocks) reaches 0.936, the highest of all conditions and 0.237 above C1's 0.699. The famine shock injected at turn 166 produces a visible dip in C1's community-reward signal (\cref{fig:reward}) and a lasting downward divergence in its cooperation trajectory: before the shock, C1 and C5 track together, but afterward C5 pulls steadily ahead (\cref{fig:cooperation}). The agents remain robust in that cooperation keeps rising rather than collapsing --- the shock\_resilience coefficient increases under pressure and the local Q-updates correct stale values --- but they do not recover the lost ground within the session. Shocks are thus the largest single environmental drag on cooperation in our study.

\subsection{Finding 4: MDP Structure}

C4 (Flat MDP) reaches 0.871, the second-highest condition (behind only C5). Without hierarchical stage gates, quest completion is faster (the bot cycles through more episodes in 500 turns), giving NPCs more learning opportunities. This suggests that quest structure affects the \textit{rate} of learning more than the \textit{direction} --- both converge toward cooperation, but flat MDPs provide more training data per unit time.

\subsection{Finding 5: Dynamic Lambda Amplifies Cooperation}

C7 (Static $\lambda = 0.325$) achieves a cooperation index of 0.595, compared to C1's 0.699 --- a 0.104 difference, making the dynamic $\lambda$ feedback loop the third-largest contributor to cooperation, after RL itself ($+0.699$) and role masking ($+0.161$). Crucially, C7 still shows strong cooperation emergence (0.000 $\to$ 0.595): the community reward signal alone drives most of the learning, even without the amplifying feedback loop. But the virtuous cycle --- positive community outcomes $\to$ cooperation rises $\to$ $\lambda$ increases $\to$ more weight on community reward --- provides a meaningful additional acceleration. This confirms that the dynamic $\lambda$ mechanism is not merely cosmetic; it is a measurable amplifier of the system's cooperation dynamics.

\chapter{Conclusion and Future Scope}
\label{chap:conclusion}

This project demonstrates that emergent cooperative behavior can arise from independent Q-learning agents when the reward signal includes a community welfare component gated by a dynamic $\lambda$ coefficient. The key findings:

\begin{enumerate}
    \item \textbf{Cooperation emerges from learning.} $\mathcal{K}(t)$ rises from 0.000 to 0.699 over 500 turns with Q-learning, while schedule-only NPCs show no change ($+0.699$ gap). This confirms the research hypothesis and makes RL by far the dominant driver.
    \item \textbf{Role masking provides the strongest inductive bias.} A 0.161 improvement in cooperation when NPCs have role-constrained action spaces, producing cleaner learning signals --- the largest single-component contribution after RL itself.
    \item \textbf{Shocks impose a lasting cost, yet agents keep learning.} Removing shocks raises final cooperation by 0.237 (C5 reaches 0.936 vs.\ C1's 0.699); the famine at turn 166 lastingly depresses C1's trajectory. The agents do not collapse --- cooperation keeps rising afterward --- but they do not recover the lost ground within the session.
    \item \textbf{The dynamic $\lambda$ feedback loop is a measurable amplifier.} Static $\lambda$ still produces strong emergence (0.000 $\to$ 0.595), but dynamic $\lambda$ adds $+0.104$ through the virtuous cycle --- the third-largest single-component contribution, after RL and role masking.
    \item \textbf{LLM narration adds quality without losing context.} With tuned prompts, the LLM achieves higher contextual grounding than templates while producing 2.6$\times$ richer vocabulary.
\end{enumerate}

\section{Limitations}
\label{sec:limitations}

\begin{itemize}[nosep]
    \item Results are from automated bot playthroughs with a single strategy (\texttt{quest\_focused}); human player behavior may produce different cooperation dynamics.
    \item The adaptation rate (0.018) and role-specific multipliers were empirically tuned; systematic sensitivity analysis is needed.
    \item Tabular Q-learning limits scalability --- 225 states is manageable for 6 NPCs but would not scale to larger worlds without function approximation.
    \item Narrative coherence metrics rely on token overlap rather than semantic similarity; sentence-transformer embeddings would provide stronger signal.
\end{itemize}

\section{Future Scope}
\label{sec:future-scope}

\begin{itemize}[nosep]
    \item Scale to 100 bot playthroughs per condition with varied strategies for statistical significance.
    \item Conduct user studies (20 participants per condition) with post-session questionnaires.
    \item Compare tabular Q-learning vs.\ DQN for NPC behavior.
    \item Extend to branching quest graphs with multiple concurrent quest lines.
    \item Explore longer sessions ($>$1000 turns) to test cooperation saturation.
\end{itemize}

\section{Real-World Transfer}
\label{sec:real-world-transfer}

The decomposed reward framework $R_i(t) = P_i(t) + G_i(t) + \lambda_i(t) \cdot C(t)$ generalises beyond games to any domain where heterogeneous subsystems must balance local objectives against a shared global target.

\textbf{Smart Factory Coordination.}
In an automated production line, each cell (stamping, welding, painting) optimises a local KPI---throughput, defect rate, idle time---analogous to role-specific $G_i$ weights. The community signal $C(t)$ maps to plant-wide OEE (Overall Equipment Effectiveness). Supply-chain disruptions, equipment failures, and demand spikes correspond directly to famine, plague, and trade-boom shocks. Current Industry~4.0 approaches rely on centralised MPC controllers that must be re-modelled when the line changes; a dynamic-$\lambda$ architecture lets each cell run a lightweight learner and join the reward structure without replanning the plant. Our C1~vs.~C3 result ($+0.699$ cooperation from RL over fixed schedules) suggests learned policies would significantly outperform static dispatch rules.

\textbf{Automotive ECU Self-Tuning.}
A modern engine's sensor--actuator subsystems (knock sensor, O$_2$ sensor, throttle body, MAP, coolant temperature, crank position) map onto heterogeneous NPC roles: each maximises a local objective (detonation margin, fuel-trim accuracy, airflow response) while the community target $C(t)$ is engine-level efficiency (e.g.\ brake-specific fuel consumption). The dynamic $\lambda$ feedback loop enables continuous self-calibration: a factory base map provides the pre-trained Q-table, and on-road experience adjusts cooperation tendencies as altitude, fuel quality, and component wear shift the reward landscape. Component aging becomes a slow-onset shock; aftermarket modifications (larger turbo, free-flowing exhaust) reset part of the state, and the system re-converges without manual tuning---our C4 result ($\mathcal{K} = 0.871$) suggests rapid re-convergence is achievable.

\textbf{Bridging the Gap.}
Two challenges must be addressed: (i)~the 225-state tabular formulation must give way to function approximation (DQN or policy gradients) for high-dimensional, real-time sensor streams; and (ii)~the $P_i$ penalty discourages but does not formally prevent catastrophic states---a hybrid architecture enforcing hard constraints from classical control theory while allowing RL-based optimisation within the safe envelope is the likely path forward.
